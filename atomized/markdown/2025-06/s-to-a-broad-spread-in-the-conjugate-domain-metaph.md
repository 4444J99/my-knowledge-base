<div>  •  The Halting Problem (Domain: Computer Science/Mathematical Logic) – Proven by Alan Turing in 1936, it states that there is no general algorithm that can determine for every possible program and input whether the program will eventually halt (stop running) or loop forever. This is a cornerstone result in computability theory. Example: If one tries to write a universal piece of software HALT(P,I) that analyzes an arbitrary program P with input I and infallibly predicts whether P halts or not, Turing showed this leads to a contradiction – essentially by constructing a program that says “if you predict I halt, I will go into an infinite loop; if you predict I loop, I will halt,” a self-referential trick. Implications: The halting problem is analogous to Gödel’s incompleteness but in computation: there are limits to what is algorithmically decidable. It reveals an inherent limit of computation – some questions are undecidable by any computer. This result launched the field of computability and complexity theory.
