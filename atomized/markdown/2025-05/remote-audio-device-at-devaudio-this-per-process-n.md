Such design shows how flattening multiple sources into one namespace can simplify access and avoid duplication of naming domains. • Semantic and Human-Friendly Names: Operating systems typically allow long, descriptive filenames but enforce rules for compatibility. Guidelines (often shared with archival practices) include using only safe characters, keeping names short, and embedding metadata like dates if needed. Filenames should be persistent, unique, and not tied to location  – for example, avoid using a folder name or device name as part of a file’s identifier, since those contexts can change in a system reboot or migration.</div>
<div>In summary, OS-level strategies for canonical naming involve hierarchy + uniqueness (no two files in the same path), and sometimes automatic versioning as an architecture (as with VMS). The Plan 9 example illustrates a radical unification, where the naming architecture itself can merge multiple systems into one flat space. MythOS can draw on these ideas by assigning each item a unique path/ID and potentially using version suffixes or similar constructs to maintain history in names.</div>
