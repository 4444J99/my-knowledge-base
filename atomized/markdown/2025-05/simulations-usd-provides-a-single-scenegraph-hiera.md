# simulations. USD provides a single scenegraph (hierarchical namespace of objects

## Context
From document: Transforming a Fragmented Digital Life into a Unified Canonical Architecture — chunk 10 (chunk 10/37)

## Content
simulations. USD provides a single scenegraph (hierarchical namespace of objects) into which multiple files and asset layers can be composed  . For example, a large virtual world might be built by many artists – one creates terrain, another buildings, another characters – each working on separate files. USD allows these to be referenced into one master scene, with composition operators that handle overrides, variants, and layering. A “variant” in USD could be, say, two versions of a building (intact and damaged) and you can switch which is active without removing the other. The scenegraph unifies all assets into a single namespace (so there’s one canonical path to each object in the world), but the system retains knowledge of what came from where via layers. This approach to aggregate multiple assets into a single view while preserving versioned differences is exactly the kind of recursive, flattened architecture MythOS aspires to. We can imagine a MythOS “world” where all personal projects are like layers in one scene – they share a common ontology of objects, but you can drill down into a specific layer or switch variants (e.g. older versions) as needed. • Snapshot and Timeline
