## Content
<div>All procedural and creative workflows are grounded in a set of foundational components that make up the *operating system* for creativity. We define these core modules explicitly within a directory structure rooted at **$OS_ROOT**. Each component corresponds to a key aspect of the OS (conceptual, memory, interaction, etc.), and together they enable the self-evolution and governance of the whole system. The primary components under $OS_ROOT are:</div>
<div><br></div>
<div>- **$OS_ROOT/config:** This module holds configuration files and global definitions that govern the OS. It includes environment variables (like paths $ARCHIVE_ROOT, $REPO_ROOT, $ORG_ID, etc.), system-wide settings, and the ontological naming rules schema. For instance, a file `$OS_ROOT/config/naming_rules.txt` might define the grammar for file naming (the LAYER-DOMAIN-TYPE-ID-VERSION scheme described later), and `$OS_ROOT/config/roles.yaml` might list the mythic archetypes assigned to various agent roles (e.g., *Oracle = Planning agent*, *Archivist = Archive agent*). This config is expected to change gradually over time as the system evolves – e.g., new environment variables might be added, or naming conventions refined – but such changes are deliberate and logged (possibly via the Handoff Seal process). Typically, only a high-level *System Manager* AI role or a human architect should have write access here, since these are the rules of the whole OS. All agents have read access to config to know the ground rules.</div>
