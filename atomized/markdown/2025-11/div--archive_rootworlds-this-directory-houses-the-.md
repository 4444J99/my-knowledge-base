## Content
<div>- **$ARCHIVE_ROOT/worlds/** – This directory houses the historical record of each world or project’s evolution. There may be subdirectories per world (for large worlds) or single files per world if they’re smaller in scope. For instance, `$ARCHIVE_ROOT/worlds/World-Alpha/` could contain files like `world_timeline.md`, `world_version1.yaml` etc., capturing snapshots or changes. Alternatively, each major version or phase of the world could be a separate file (e.g., `WORLD-MYTH-LOG-ALPHA-v1_0.md` might log all mythological expansions to World Alpha in version 1.0). The **index** for worlds (perhaps `$ARCHIVE_ROOT/worlds/index.txt` or a structured file) enumerates each world by ID, name, and provides the path to its archive records and a brief description. Within each world’s archive, we also maintain cross-links to related archives, like references to session logs where that world was discussed, or links to character files in the characters archive.</div>
<div>- **$ARCHIVE_ROOT/characters/** – Because characters (or agents, or key entities) can exist across worlds or have their own development arcs, we dedicate a section of the archive to them. Each character that reaches a certain level of importance (say a main character in a narrative, or a principal AI agent persona) gets an entry in this directory. For example, a character named “Aster” with ID 0420 might have an archive file `CHAR-0420_Aster.md` containing their origin, development notes, and references to appearances in stories or projects. This is beneficial for mythic consistency: if a character recurs or if we fuse mythologies across worlds, we have a single point of truth for that character’s lore. The character archive index (`$ARCHIVE_ROOT/characters/index.txt`) would list each character ID, name, archetype (if applicable), and link to their file, along with tags for the world or project they belong to. This way, future agents can quickly retrieve a character’s backstory without scanning every world file.</div>
