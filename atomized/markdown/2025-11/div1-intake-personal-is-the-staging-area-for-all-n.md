## Content
<div>1. INTAKE: Personal is the staging area for ALL new ideas</div>
<div>2. EVALUATION: Repos stay personal until they prove value</div>
<div>3. CRITERIA: Only graduate to org when:</div>
<div>   - You've made substantial original modifications</div>
<div>   - It's actively used in a workflow</div>
<div>   - It solves a specific system need</div>
<div>   - It integrates with other org components</div>
<div><br></div>
<div>## Organization Rules (ivi374forivi)</div>
<div>1. PURPOSE: Org contains only intentional, maintained projects</div>
<div>2. STATUS: Every repo must have clear status label:</div>
<div>   - [PRODUCTION] - Live and maintained</div>
<div>   - [DEVELOPMENT] - Active work in progress</div>
<div>   - [EXPERIMENTAL] - Exploratory, may be abandoned</div>
<div>   - [DEPRECATED] - No longer maintained</div>
<div>3. DOCUMENTATION: Every org repo requires:</div>
<div>   - Clear purpose statement</div>
<div>   - Integration points with other repos</div>
<div>   - Current status and next steps</div>
<div><br></div>
<div>## Migration Rules (Personal → Org)</div>
<div>1. CHECKLIST before migration:</div>
<div>   - [ ] Original work committed (not just a fork)</div>
<div>   - [ ] README explains purpose and integration</div>
