# <div>  - Configure repository settings (Private/Public).</div>

## Context
From document: full best practices house keeping; — chunk 16 (chunk 16/24)

## Content
<div>  - Configure repository settings (Private/Public).</div>
<div>- [ ] Add a `README.md` file for each repository.</div>
<div>- [ ] Add a `.gitignore` file (choose language-specific templates).</div>
<div>- [ ] Set up a **LICENSE** for the repository.</div>
<div>- [ ] Enable branch protection rules:</div>
<div>  - Go to **Settings** &gt **Branches** &gt Add **protection rules** for `main` or `default` branches.</div>
<div>- [ ] Add repository collaborators and assign permissions.</div>
<div><br></div>
<div>---</div>
<div><br></div>
<div>### **3. Set Up GitHub Projects**</div>
<div>- [ ] Create a **Project** board:</div>
<div>  - Go to **Projects** &gt **New Project**.</div>
<div>  - Choose between **Kanban** or **Table** layout.</div>
<div>- [ ] Add columns to the project board:</div>
<div>  - Example: **To Do**, **In Progress**, **Done**.</div>
<div>- [ ] Link repositories to the project board.</div>
<div>- [ ] Automate workflows (e.g., automatically move cards when issues are closed).</div>
<div><br></div>
<div>---</div>
<div><br></div>
<div>### **4. Configure Issues**</div>
<div>- [ ] Enable Issues in repository settings.</div>
<div>- [ ] Set up issue templates:</div>
<div>  - Add `.github/ISSUE_TEMPLATE` directory.</div>
