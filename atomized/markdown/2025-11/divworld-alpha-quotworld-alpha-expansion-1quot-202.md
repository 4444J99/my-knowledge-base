<div>The **Digital Memory Archive** is append-only and versioned: if something must be changed (say a correction in a spec), we create a new entry (and possibly mark the old one as superseded). This ensures an accurate historical record. By designing the archive this way, we empower future AI collaborators to answer questions like “How did we arrive at this design decision?” by looking at the spec and session logs around that decision, or “Has this character appeared before?” by checking the character index. There’s no reliance on querying an external database or vector memory; the knowledge *is the file system*. This simplicity is robust: plain text persists across systems and is diff-able, searchable, and easy to sync (with Git or other tools). The archive is essentially the **memory palace** of the OS, with each room (folder) and scroll (file) carefully labeled for eternity.</div>
