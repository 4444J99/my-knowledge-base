<div>  Type helps in filtering by what something is. For example, `INDEX` files can be easily identified to know they are navigational aids. `PROMPT` files are clearly meant to be used in AI prompting, etc.</div>
<div>  </div>
<div>- **ID:** The identifier is a short symbolic tag unique to the artifact within its scope, often a human-meaningful code or a number (or combination). It may sometimes combine a code and number. For example:</div>
<div>  - Could be a short word or abbreviation plus a numeric sequence (like `LANG-0001` to indicate maybe a language concept entry, or `CHAR-0420` as a character ID shown in the earlier example).</div>
<div>  - For worlds or projects, it might be the world’s code (e.g., `ALPHA`, `BETA`).</div>
<div>  - If the artifact is specific to a world, the world’s ID might be part of it (like `ALPHA-` prefixing an ID).</div>
<div>  - The scheme is flexible: the main point is that each artifact gets a unique handle that can be referenced elsewhere. We might use zero-padded numbers for sorting (e.g., 0001, 0002, etc.), especially in a series.</div>
<div>  The ID should be meaningful enough to hint at content but not too verbose. For instance, `OS-0001` might refer to an overarching OS spec (like this one), whereas `CHAR-0420` uniquely picks out a character.</div>
