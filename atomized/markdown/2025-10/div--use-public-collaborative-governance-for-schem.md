<div>- Use public, collaborative governance for schema decisions; involve stakeholders in discussions</div>
<div>- Create comprehensive documentation showing real-world examples and use cases</div>
<div>- Maintain experimental branches for proposed changes before committing them to stable versions</div>
<div>- Build tools that make it easy for developers to validate and implement your standards</div>
