# <div>- [ ] Enable Issues in repository settings.</div>

## Context
From document: ### Checklist for Setting Up a GitHub Organization and Repository — chunk 3 (chunk 3/5)

## Content
<div>- [ ] Enable Issues in repository settings.</div>
<div>- [ ] Set up issue templates:</div>
<div>  - Add `.github/ISSUE_TEMPLATE` directory.</div>
<div>  - Create issue templates (e.g., Bug Report, Feature Request).</div>
<div>- [ ] Add labels for issues:</div>
<div>  - Examples: `bug`, `enhancement`, `help wanted`, `good first issue`.</div>
<div>- [ ] Enable **Discussions** for community engagement.</div>
<div><br></div>
<div>---</div>
<div><br></div>
<div>### **5. Set Up GitHub Actions**</div>
<div>- [ ] Enable Actions in repository settings.</div>
<div>- [ ] Add workflow files:</div>
<div>  - Create `.github/workflows` directory.</div>
<div>  - Add YAML files for CI/CD workflows (e.g., `build.yml`, `test.yml`, `deploy.yml`).</div>
<div>- [ ] Use pre-built Actions from the GitHub Marketplace.</div>
<div>- [ ] Set up secrets for workflows:</div>
<div>  - Go to **Settings** &gt **Secrets and variables** &gt **Actions**.</div>
<div>  - Add sensitive credentials (e.g., API keys).</div>
<div><br></div>

<div>### **6. Configure Security and Compliance**</div>
